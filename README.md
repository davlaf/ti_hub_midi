# ti_hub_midi
Midi for TI Innovator Hub

# Demos

If you are here just to listen to the demos, go into the demo folder, download the .tns files, open them in the CX CAS software and run them. There is no need for all the complicated steps below, they are the instructions on how to make new songs.

# Requirements

You will need:
- Python 3
- pretty_midi python library (`pip install pretty_midi`)
- A TI Innovator Hub
- TI-Nspire CX CAS software

[TI Innovator Hub photo](https://i5.walmartimages.com/asr/5f163297-ba37-47a0-b602-2ab56a0bde79_1.c9d15abed6b2c994329e82aa1666eb9c.jpeg "A TI Innovator Hub")

# How to use

1. Find a midi file online
2. Open it in audacity (or other midi software) to identify the tracks you want to remove.
    - This is to remove drums and reduce the number of notes being played, as too many notes will make the output sound bad
3. Open the **Midi to TI Hub convert.py** in a text editor and edit this line to add the instruments you want to remove.
    ```python
    INSTRUMENT_OMIT_LIST = [] 
    ```
    You can use the first few lines of code output to help. In this case, we want to remove tracks 3 and 4
    
    *In audacity, the track number may not be the same as in this script but the order is identical. For example, if in audacity track number 1 5 6 and 9 show up and you want to disable track 5 and 9, you would write [2,4] because they are the second and fourth track in order.*
    ```
    1: "Lead"
    2: "Bass"
    3: "Hi-Hat"
    4: "Kick"
    ```
    So in this example, we would write this:
    ```python
    INSTRUMENT_OMIT_LIST = [3,4]
    ```
    Or this:
    ```python
    INSTRUMENT_OMIT_LIST = ["Hi-Hat","Kick"]
    ```
5. Edit the the **Midi to TI Hub convert.py** file and add the name of the midi file by writing it to this line:
    ```python
    MIDI_FILE_NAME = ""
    ```
    For example:
    ```python
    MIDI_FILE_NAME = "fur_elise.mid"
    ```
6. Save the modifications and run the program.
7. Copy the output in curly brackets
8. Plug in the TI Innovator hub, go into the CX CAS software, and open the **Midi player.tns** project.
9. Paste the previous output into in this line after:
    ```python
    note_dict = #has fur elise by default
    ```
10. Press CTRL+R to listen!

# Settings

In the **Midi to TI Hub convert.py** file you can change this variable.
```python
DIVISION = 0.05
```
This is the interval that the program adds notes to the note dictionary.
For example, a value of 0.05 means that every 0.05 seconds of the song, the notes being played are added to the dictionary. This cuts the song down to a format the TI Hub can play.
If the notes are long and the song is slow, increase this number. If the notes are short and the song is fast, decrease this number. (only change it if there are issues, because 0.05 works fine for most songs)

In the **Midi player.tns** project, there are two variables to change.

```python
FLASH_LENGTH = 0.01
```

```python
FLASHES_PER_DIVISION = 2
```
Because the TI Hub can only play one note at once, to simulate multiple being played the code plays (or flashes) the notes quickly to create the illusion multiple are being played at once.

The **FLASH_LENGTH** variable determines how long a note will be played when it is being flashed. Too high and the illusion of multiple notes playing is broken, and the song gets played very slowly. Too low and the notes will sound distorted. A good sweet spot is between 0.025 and 0.01. If the song is playing too slowly or too fast this is the variable you want to change first. 

The **FLASHES_PER_DIVISON** variable determines how many notes are played per division of the song. This means that if it is set to 2 and a "note group" (division) has more than 2 notes in it, they will be skipped. If there is only one note, it will be played twice. This variable should be changed to accomodate for the maximum number of notes the song has at once, but if it slows down the song too much it can be reduced (I did this for the fur elise example).
